# PWA + Offline Support — Design

**Date:** 2026-06-09
**Status:** Approved (pending spec review)

## Goal

Make "Guess the Winrate" installable as a PWA and fully playable offline. The
app is a static React SPA whose entire dataset (`src/data/openings.json`, 1.3 MB)
is bundled into the JS, so once the build assets are cached the game needs no
network at all.

## Approach

Use **`vite-plugin-pwa`** (Workbox-based) rather than a hand-rolled service
worker. The plugin auto-generates the precache manifest from build output
(handling hashed filenames), respects Vite's `base` path, generates the web app
manifest, and provides an update helper (`virtual:pwa-register`). A hand-rolled
SW would require manual cache-list maintenance and update logic — more
error-prone for no benefit.

Trade-off accepted: one build-time devDependency. It does not ship in the
production bundle.

## Components

1. **`vite-plugin-pwa`** added to `vite.config.ts` with `registerType: 'prompt'`
   (toast-driven updates, not silent auto-update).
2. **Web App Manifest** (generated by the plugin):
   - `name` / `short_name`: "승률 맞히기" / "Guess the Winrate"
   - `theme_color`: `#2a2a2a`, `background_color`: `#2a2a2a`
   - `display: standalone`
   - `start_url` and `scope`: the Vite base `/guess-the-winrate/`
   - `lang: ko`
3. **PWA icons** — generated from existing `public/favicon.svg` (chessboard) via
   `@resvg/resvg-js-cli` (same tool used for `og.png`):
   - `pwa-192.png` (192×192)
   - `pwa-512.png` (512×512)
   - `pwa-maskable-512.png` (512×512, with safe-zone padding, `purpose: maskable`)
   - Source SVGs live under `scripts/`.
4. **Service Worker** (Workbox `generateSW`) — precache app shell + JS + CSS +
   icons. Because `openings.json` is part of the JS bundle, precaching the
   bundle covers the full dataset → complete offline play.
5. **Update toast** — a small bottom banner shown when a new SW is waiting:
   "새 버전이 있어요 [새로고침]" / "A new version is available [Refresh]".
   Clicking calls `updateSW()` (activate new SW + reload). i18n via existing
   `strings.ts`.

## Behaviour / Lifecycle

**First visit (online):** app loads → SW registered in `main.tsx` → Workbox
precaches assets in the background → offline-ready once caching completes.

**Return visit / offline:** SW serves from cache instantly; game works with no
network (questions, win rates, chessboard all bundled).

**After a new deploy:** on next open, the new build is detected and installed as
a "waiting" SW → `onNeedRefresh` fires → update toast shown → user clicks
Refresh → `updateSW()` activates the new SW and reloads. If ignored, the old
version keeps working (no mid-game screen swap — the intended safe UX).

## Edge Cases & Decisions

- **GitHub Pages subpath:** SW served from `/guess-the-winrate/sw.js`, scope
  limited to `/guess-the-winrate/`. Correct and intended (no interference with
  other project pages on the same `hdh4952.github.io` origin).
- **Install prompt:** no custom "Install" button (YAGNI). Rely on the browser's
  native install affordance (address-bar icon / mobile "Add to Home Screen").
- **GoatCounter analytics:** external script (`gc.zgo.at`) is NOT precached; it
  fails silently offline. Analytics only when online — acceptable.
- **localStorage** (language, best score): independent of the SW, unchanged.
- **Dev mode:** SW active only in production builds (plugin default) to avoid
  cache confusion during development.

## Testing

- **Unit (vitest):** update-toast component — renders on `onNeedRefresh`,
  calls `updateSW` on button click, shows correct i18n copy (ko/en).
  `virtual:pwa-register` is mocked.
- **Manual:** `npm run build && npm run preview` → DevTools › Application:
  (1) manifest recognized, (2) SW registered, (3) toggle Network "Offline" and
  reload → game still works, (4) install available.
- All existing 72 tests must continue to pass.

## Out of Scope

- Custom install button / install analytics
- Runtime caching of third-party requests (analytics)
- Bundle splitting of `openings.json` (separate future task; if done later, the
  split data file must be added to the Workbox precache list to preserve offline
  play)
- Push notifications, background sync
