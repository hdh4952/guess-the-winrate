# 오프닝 승률 맞추기 게임 — 설계 문서

작성일: 2026-06-08

## 1. 개요

두 개의 체스 오프닝을 보여주고, 선택한 레이팅 구간에서 **어느 오프닝의 승률이 더 높은지** 맞추는 웹 게임. 승률 비교는 매 라운드 무작위로 정해진 **백 관점** 또는 **흑 관점** 한쪽 기준으로 한다.

데이터는 Lichess Opening Explorer API에서 가져온다.

## 2. 핵심 규칙

- 사용자는 게임 시작 시 **레이팅 구간**을 하나 선택한다.
- 매 라운드 **6수(플라이 ≥ 6, 즉 백·흑 각 3수) 이상 진행되고 이름이 붙은 서로 다른 오프닝 2개**를 출제한다.
- 매 라운드 **관점(백/흑)을 무작위로** 정한다. 두 오프닝 모두 같은 색 관점에서 비교한다.
  - 백 관점: `white / (white + draws + black)`
  - 흑 관점: `black / (white + draws + black)`
- 사용자가 승률이 더 높은 오프닝을 고르면 정답.

## 3. 기술 스택

- **React + Vite + TypeScript**, 순수 클라이언트 SPA (백엔드 없음).
- 체스판 렌더링: `react-chessboard` (정적, 비대화형).
- 수순 파싱/FEN 계산: `chess.js`.
- 테스트: `Vitest`.
- 정적 호스팅 가능.

## 4. 데이터 소스

### 4.1 오프닝 사전 (빌드 시 번들)

- 소스: `lichess/chess-openings` 저장소의 `a.tsv`~`e.tsv` (각 행: `eco`, `name`, `pgn`).
- 변환 스크립트 `scripts/build-openings.ts`:
  - PGN → 수순 파싱.
  - **플라이 ≥ 6** 항목만 유지.
  - 정규화하여 `src/data/openings.json` 생성. 각 항목 형태:
    ```ts
    interface OpeningEntry {
      eco: string;        // 예: "C50"
      name: string;       // 예: "Italian Game: Giuoco Piano"
      sanMoves: string[]; // 예: ["e4","e5","Nf3","Nc6","Bc4","Bc5"]
      uciMoves: string[]; // 예: ["e2e4","e7e5","g1f3","b8c6","f1c4","f8c5"]
      fen: string;        // 최종 위치 FEN
    }
    ```

### 4.2 Lichess Opening Explorer API (런타임)

- 엔드포인트: `https://explorer.lichess.ovh/lichess`
- 쿼리: `variant=standard`, `play=<uciMoves 콤마구분>`, `speeds=blitz,rapid,classical`, `ratings=<bucketStart>`
- 응답에서 사용하는 필드: `white`(백 승 수), `draws`(무 수), `black`(흑 승 수).
- 총 게임 수 = `white + draws + black`.

### 4.3 레이팅 구간 매핑

Lichess 버킷 시작값으로 매핑한다.

| 표시 라벨 | API `ratings` 값 |
|---|---|
| 0-1000 | 0 |
| 1000-1200 | 1000 |
| 1200-1400 | 1200 |
| 1400-1600 | 1400 |
| 1600-1800 | 1600 |
| 1800-2000 | 1800 |
| 2000-2200 | 2000 |
| 2200-2500 | 2200 |
| 2500+ | 2500 |

## 5. 게임 흐름 / 라운드 생명주기

1. **시작 화면**: 레이팅 구간 선택 → 게임 시작.
2. **라운드 생성** (`useRound` 훅):
   1. `openings.json`에서 서로 다른 오프닝 2개를 무작위 추출.
   2. 각 오프닝에 대해 Explorer API 호출 (선택 구간 기준).
   3. 둘 중 하나라도 **총 게임 수 < 최소 임계값(기본 1000)** 이면 해당 오프닝 재추출 후 재호출.
   4. 관점(백/흑) 무작위 결정.
   5. 두 오프닝의 해당 관점 승률 차가 **0이면** 라운드 무효 → 재추출.
3. **출제 화면**: 두 오프닝 카드(체스판 + 이름 + SAN 수순)와 질문 "어느 쪽이 **[백/흑]** 승률이 더 높을까요?". 승률은 숨김.
4. **선택**: 사용자가 한 오프닝 선택.
5. **공개**: 두 오프닝의 승/무/패 막대그래프 + 승률 % + **표본(게임) 수** 표시, 정/오답 표시.
6. **스코어**: 정답 시 스트릭 +1, 오답 시 0으로 리셋. 최고 스트릭은 `localStorage`에 저장. "다음 문제"로 계속.

## 6. 컴포넌트 / 모듈

- `RatingPicker` — 레이팅 구간 선택 (시작 화면).
- `GameScreen` — 라운드 상태 관리, 두 카드 + 질문 + 선택 + 공개.
- `OpeningCard` — 체스판 다이어그램 + 오프닝 이름 + SAN 수순. 공개 시 결과 막대 표시.
- `ResultBars` — 승/무/패 비율 막대 + 승률 % + 표본 수.
- `ScoreBar` — 현재 스트릭 / 최고 기록.
- `useRound` (훅) — 오프닝 추출 · API 호출 · 재추출 · 관점 결정 캡슐화.
- `lichessApi.ts` — Explorer 호출 + 응답 캐시(같은 오프닝+구간 메모이즈) + 호출 간격 제어.
- `winrate.ts` (순수 함수) — 승률 계산, 정답 판정.

## 7. 엣지 케이스 / 리스크

- **CORS**: `explorer.lichess.ovh`는 `Access-Control-Allow-Origin: *`를 제공하여 브라우저 직접 호출 가능. 차단 시 Vite dev proxy로 폴백.
- **레이트 리밋**: 호출 간 간격 + 응답 캐시로 완화. 라운드당 호출 2회.
- **표본 부족**: 최소 게임 수 임계값 미만이면 재추출.
- **동률 승률**: 차가 0이면 라운드 무효 후 재추출.
- **로딩/에러**: API 실패 시 재시도 버튼 및 로딩 표시.

## 8. 테스트 전략

- 순수 함수(`winrate.ts`: 승률 계산·정답 판정, 재추출 조건)는 Vitest 단위 테스트.
- `lichessApi.ts`는 fetch 모킹으로 테스트.
- 빌드 스크립트는 샘플 TSV 입력에 대해 ≥6수 필터링과 정규화 출력 검증.

## 9. 범위 외 (YAGNI)

- 힌트/난이도 옵션, 계정/리더보드, 시간제어 사용자 선택, 다국어 등은 이번 범위에서 제외.
